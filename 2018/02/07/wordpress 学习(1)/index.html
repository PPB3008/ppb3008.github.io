wordpress 学习(1)

#wordpress数据表结构
wordpress中的内容类型：
文章，页面，自定义文章类型，附件，链接，导航菜单项目（以文章方式保存）

附加在内容上的数据：
分类目录，标签，自定义分类法/分类，文章元数据

以及其他类型：
小工具，选项，用户，站点，硬编码内容（主题相关，类似hexo中的主题提供生成根基），调用的站外数据

wp内使用的表总共11个
options posts postmeta users usermeta comments comment-meta term term-taxonomy term-relationships links

post-meta关联posts
usermeta关联users
comment-meta关联comments
其中meta是关联表内容的元数据

comments关联posts posts关联users

term-relationship关联links posts term-taxonomy term-taxonomy关联terms
options独立

基本关系:三大块和他们的元数据 users posts comments和他们的meta comments依赖posts posts依赖users
两小块和posts的relationships：posts links term-taxonomy三者的关系 存在于term-relationships term-taxonomy是term的分类
一小块独立：options

#post，page和post type (?)
不限定类型，公用一个显示样式的消息/内容/帖子，均叫做post
一个post的各种不同类型即posttype，他储存在posts表里的post_type键内，
所有种类的posttype:
post 基础类型,blog常用,按时间顺序反向显示在网站中
page 像post但不受时间制约，可以用不同模板差异化构建，不能分配类别标签
attachment 保存上传到站点的文件的信息
revision
navigation menu 导航目录
custom css 自定义的post css 保存除theme css外额外的css
changesets 用于在自定义或定制时保留更改，在关闭浏览器后自动还原

post-type可以custom，自定义出一个post类型。比如使用论坛就可以自创一个发送帖子的post_type

#目录权限问题
在mac上创建pages需要上传各类media如图片视频等时，会出现目录无权限修改的问题，可以去apache目录下修改httpd.conf
在下增加
Options Indexes FollowSymLinks
AllowOverride All
Order allow,deny
Allow from all
Require all granted
这些选项，同时将wordpress/wp_content的权限改为read&write
